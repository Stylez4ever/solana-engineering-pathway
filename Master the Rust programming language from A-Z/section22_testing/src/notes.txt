An assertion
- is a verification that a statement is valid. To assert means to state a fact.

A regression
- is a bug introduced into working software
- to regress means to return to a former or less developed state

The assert not equal (assert_ne!)
- marco validates that two values are not equal.

unit tests
- a unit is a small, independent chunk of code or functionality
- view it as one small building block of a larger application
- testing an individual method is an example of a unit test
- like we are zooming the lens in on one single isolated functionality

integration tests
- are larger in scope than unit tests, they may test a feature as a whole. they may test a 
feature that will require for example, the communication or interaction of multiplie struct, 
multiple types or function working together
